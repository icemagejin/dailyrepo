### 重要经验教训

1. **记录必须清晰准确**
   - 错误的历史记录会导致后续错误
   - 需要定期审查和修正记录

2. **用户确认是关键**
   - 不应假设理解正确
   - 需要主动确认规则，特别是涉及多个存储位置的情况

3. **边界必须明确**
   - Notion 三个数据库的用途必须明确区分
   - GitHub 两个仓库的用途必须明确区分
   - 哪些自动同步，哪些需要用户指定，必须清楚

4. **错误需要立即修正**
   - 发现错误同步后立即删除和重新同步
   - 修正历史记录，避免误导后续操作
